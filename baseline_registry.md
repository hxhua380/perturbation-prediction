## 最终选择门槛

必须核实官方代码与许可、数据下载与预处理、环境依赖、入口和最小命令、论文划分与指标、单张 A100 资源、维护状态及已知 issue 后，再作主/备用决定。

## 第一轮阶段性推荐（不是最终锁定）

- **主 baseline：GEARS。** 任务与组合 CRISPR 目标一致，论文/代码/MIT 许可公开，官方 API 可直接加载 Norman 并指定 `simulation` 划分。主要风险是最后代码推送距今较久、PyG 环境、自动下载和评估设置需要审计。
- **备用 baseline：CPA。** 官方包、测试、教程和组合 CRISPR 示例比许多研究仓库完整，且能扩展到药物/剂量/上下文。缺点是研究问题更宽，对“完全未见基因”的针对性不如 GEARS。
- **简化模型：no-perturb + mean-shift/linear + additive。** 这是必要对照而非论文创新；组合任务中 additive baseline 尤其重要。

## 资源判断

- 官方示例模型 hidden size 为 64、batch size 32、20 epochs；这表明模型本体不属于大型基础模型。
- 现有一张 A100 从算力等级上预计足以运行 Norman 上的 GEARS，但**准确显存、时长和完整训练成本尚未核实**。论文/README 没有给出可直接迁移到本服务器的可靠时长，服务器显存、调度和作业限制也未知，因此当前不报具体小时数。
- 首次冒烟测试建议使用 GEARS 官方 `PertData.load('norman')` 对应的预处理数据，再从训练集做一个仅用于连通流程的小子集和 1-2 个 batch/极少 epoch。冒烟结果不能用于论文性能结论。
