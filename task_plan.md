## 关键问题

1. 哪个模型在公开性、稳定性、复现成本和改进空间之间最适合新手？
2. 第一次冒烟测试应使用哪个小型预处理数据及哪种固定划分？
3. 论文之间指标差异有多少来自任务定义、预处理或数据划分？
4. 单张 A100 和服务器限制能否支持完整复现？

## 已作决策

| 决策 | 理由 |
|---|---|
| 先骨架与来源核实，再选择 baseline | 防止被最新或最复杂模型带偏 |
| 大型数据和训练推迟到服务器审计后 | 避免磁盘、许可和资源风险 |
| 复现与改进分离，保留原 baseline | 确保结果归因清楚 |

## 错误记录

| 错误 | 尝试 | 处理 |
|---|---:|---|
| Windows 沙箱辅助程序缺失，首个只读命令未启动 | 1 | 按权限流程获得批准后重新执行，只读审计成功 |
| `git status` / `git log` 报“not a git repository” | 1 | 记录为环境事实；本轮不擅自初始化 Git |
| Web 工具拒绝直接打开 `api.github.com` 仓库元数据 URL | 1 | 改用官方 GitHub 页面和论文代码可用性声明；后续可用只读 GitHub 查询补齐更新时间 |
| PowerShell GitHub API 汇总命令出现空管道语法错误 | 1 | 将 `foreach` 输出保存到变量后再格式化，第二次成功 |
